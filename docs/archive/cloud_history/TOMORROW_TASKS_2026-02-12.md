## Phase 1 — נעילת חוזה טופס כפיר
1. לעבור סעיף-סעיף על `docs/cloud/KFIR_FORM_SPEC_DRAFT.md`.
2. לסגור סופית:
   - רשימת פריטים
   - תקן לכל פריט
   - סימון פריטים קריטיים
3. לאשר נוסח שדות טופס:
   - `תקין / חסר / תקול`
   - שדה פירוט קצר במקרה `חסר`/`תקול`.

### פלט חובה
- גרסת spec מאושרת אחת (`v1`) ללא סעיפים פתוחים.

## Phase 2 — יצירת Google Form אמיתי
1. להריץ Apps Script מתוך:
   - `docs/forms/kfir_google_form_apps_script.gs`
2. ליצור טופס שבועי פלוגת כפיר + לאמת קישור ל-Sheet תגובות.
3. לשמור מזהים:

   - Sheet ID

### פלט חובה
- טופס פעיל אחד + Sheet פעיל אחד.

## Phase 3 — חיבור קליטה למערכת
1. לעדכן env/secrets עם Sheet ID החדש.
2. לוודא שמנגנון sync/refresh עובד עם הנתונים החדשים.
3. להריץ בדיקת Smoke API:

   - `/v1/views/companies/Kfir`

### פלט חובה
- המערכת מזהה לפחות שבוע אחד חדש מתוך הטופס.
